## Q1
Object detection khác image classification ở điểm nào?

- [x] Object detection vừa phân loại (classification) vừa định vị (localization) vật thể bằng bounding box, không chỉ đưa ra 1 nhãn cho cả ảnh
- [ ] Object detection không cần dùng CNN
- [ ] Object detection chỉ hoạt động với ảnh đen trắng
- [ ] Object detection và classification hoàn toàn giống nhau

> Classification chỉ trả lời "ảnh này có gì", object detection trả lời thêm "vật thể đó nằm ở đâu" — dữ liệu train cần cả nhãn lớp và toạ độ bounding box, nhiều thông tin hơn classification thuần.

## Q2
IoU (Intersection over Union) đo lường điều gì, và giá trị của nó nằm trong khoảng nào?

- [x] Tỷ lệ diện tích giao nhau trên diện tích hợp của 2 bounding box, luôn nằm trong [0, 1]
- [ ] Khoảng cách Euclid giữa tâm 2 bounding box
- [ ] Số lượng vật thể trong ảnh
- [ ] Độ tự tin (confidence score) của một dự đoán

> IoU = 1 nghĩa là 2 box trùng khớp hoàn hảo; IoU = 0 nghĩa là không chồng lên nhau. Trong thực hành, một dự đoán thường coi là "đúng" nếu IoU với box thật vượt ngưỡng (thường 0.5).

## Q3
Vì sao mô hình object detection dùng anchor box thay vì để mô hình tự do dự đoán toạ độ bounding box từ đầu?

- [x] Dự đoán tự do toạ độ box rất khó huấn luyện ổn định; anchor box cho sẵn các khung tham chiếu, mô hình chỉ cần dự đoán lớp + offset (độ lệch) để tinh chỉnh
- [ ] Anchor box giúp giảm số lượng lớp cần phân loại
- [ ] Anchor box chỉ dùng cho ảnh có kích thước cố định
- [ ] Anchor box thay thế hoàn toàn nhu cầu dùng CNN

> Anchor box là các bounding box định trước với nhiều scale/aspect ratio tại mỗi vị trí — mô hình dự đoán lớp và offset (độ lệch so với anchor) thay vì toạ độ tuyệt đối, dễ huấn luyện ổn định hơn.

## Q4
Non-Maximum Suppression (NMS) giải quyết vấn đề gì?

- [x] Lọc bỏ các dự đoán trùng lặp (nhiều anchor box cùng "chỉ" vào một vật thể), chỉ giữ lại dự đoán có độ tự tin cao nhất cho mỗi vật thể
- [ ] Tăng số lượng bounding box được dự đoán
- [ ] Chuyển object detection thành bài toán segmentation
- [ ] Huấn luyện lại toàn bộ mô hình từ đầu

> NMS sắp xếp dự đoán theo confidence giảm dần, chọn dự đoán cao nhất, loại bỏ mọi dự đoán khác có IoU vượt ngưỡng với nó, lặp lại — kết quả là một tập box nhỏ, mỗi box đại diện đúng 1 vật thể thay vì hàng chục box chồng chéo.

## Q5
Fast R-CNN cải tiến gì so với R-CNN gốc?

- [x] Chạy CNN một lần duy nhất trên toàn bộ ảnh (dùng RoI Pooling để trích đặc trưng từng vùng từ bản đồ chung), thay vì chạy CNN riêng biệt cho từng vùng đề xuất
- [ ] Fast R-CNN bỏ hoàn toàn bước đề xuất vùng, dự đoán trực tiếp như YOLO
- [ ] Fast R-CNN không cần huấn luyện, chỉ dùng luật cố định
- [ ] Fast R-CNN chỉ hoạt động với ảnh xám, không hoạt động với ảnh màu

> R-CNN chạy CNN độc lập trên ~2000 vùng đề xuất mỗi ảnh — rất nhiều tính toán lặp lại. Fast R-CNN giải quyết đúng bottleneck đó: CNN chạy 1 lần trên toàn ảnh, RoI Pooling trích đặc trưng từng vùng từ bản đồ chung. Faster R-CNN sau đó giải quyết bottleneck còn lại (selective search) bằng Region Proposal Network.
